## Key Decisions

**Thought partner model — negotiated intelligence**
- **Decided:** The app should reconcile Claude's read with the user's read, not just present synthesis as output
- **Why:** Claude only knows what was said in the room. The user has context that never makes it into transcripts. The most valuable signal is where the two diverge.
- **Resolution options:** Accept, Edit, Remove, Hypothesise (testable belief with test condition), Annotate
- **Assumes:** Annotation schema is designed before desktop app is built — this is a breaking data model change
- **Where it lives:** `context-library/architecture-decisions.md` §5, not yet implemented

**Desktop app removes Cloudflare Worker**
- **Decided:** Desktop app (Tauri) calls Anthropic API directly — no proxy needed. API key stored in OS keychain.
- **Why:** Native apps have file system access and can hold secrets securely. Eliminates Worker deployment complexity for real users.
- **Web demo stays:** GitHub Pages + Cloudflare Worker remains for public demo/sharing. Two separate deployment targets.
- **Rejected alternatives:** Building the thought partner UI in the terminal (impossible for visual annotation work)
- **Where it lives:** `context-library/architecture-decisions.md` §4

**Projects as records, not folders**
- **Decided:** A project should be a registry entry linking to multiple folder locations, not assumed to be a single folder
- **Why:** Sobremesa exists in both `projects/sobremesa/` (PM work) and `stakeholder-intelligence-app/projects/sobremesa/` (stakeholder intelligence). No single place currently says "here is everything about Sobremesa."
- **Proposed solution:** `projects-registry.md` at root level. Each project = record with folder links, people, status.
- **NOT doing yet:** Do not restructure folders until registry is built and validated.
- **Where it lives:** `context-library/architecture-decisions.md` §3

**Global people graph at root**
- **Decided:** Stakeholder profiles and memory belong at root `context-library/people/`, not inside project folders
- **Why:** People exist across projects. Marcus Webb's Talent Bridge context is relevant when he appears in Sobremesa. Per-project profiles lose cross-project patterns.
- **Synthesis as quality gate:** `/_project-synthesis` decides what promotes to global memory — not every meeting note.
- **Where it lives:** `context-library/architecture-decisions.md` §2, not yet implemented
