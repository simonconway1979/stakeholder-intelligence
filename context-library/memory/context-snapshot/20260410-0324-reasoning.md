## What We Tried and Abandoned

**Desktop app as primary interface**
- **Tried:** Planned a Tauri/Electron desktop app to handle transcript imports and viewing
- **Why abandoned:** User clarified target users are Claude Code PMs already in the terminal. A desktop app adds installation complexity without adding capability for this user profile.
- **What we learned:** The friction is the import UX (multi-step CLI is fine), not the view UX (web app is already good). Desktop app stays on backlog for a different target market.

**`wrangler secret put` via CLI**
- **Tried:** Multiple approaches to set the API key via wrangler CLI (`!` prefix, echo pipe, regular terminal)
- **Why abandoned:** All approaches hit "non-interactive context" error. Wrangler requires a TTY for secret input.
- **What we learned:** Cloudflare dashboard is the reliable path for secret management. Note in CHAT-SETUP.md reflects this.

**API key accidentally set as a plain variable**
- **Tried:** User set `ANTHROPIC_API_KEY` as a `var` in the Cloudflare dashboard rather than a `Secret`
- **Why abandoned:** Key was exposed in plain text in the wrangler deploy warning output (and therefore in this conversation)
- **What we learned:** Always use Secrets (encrypted) not Variables (plain text) for API keys. Key was rotated. CHAT-SETUP.md explicitly says "choose Secret, not Variable."
