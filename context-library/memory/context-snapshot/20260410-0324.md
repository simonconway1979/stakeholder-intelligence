## Key Decisions

**Cloudflare Worker for API proxy (not Claude.ai managed hosting)**
- **Decided:** Cloudflare Worker proxies Anthropic API calls; key stored as Cloudflare secret
- **Why:** Free tier (100k req/day), project data embedded in worker for prompt caching, API key never touches browser, works with GitHub Pages static hosting
- **Rejected alternatives:** Claude.ai managed hosting (not available in the way the user thought); sending data from client each request (kills caching benefit)
- **Assumes:** User has a Cloudflare account and is comfortable with wrangler CLI
- **Where it lives:** `worker.js`, `wrangler.toml`, `CHAT-SETUP.md`

**Terminal flow over desktop app**
- **Decided:** Improve `/_add-transcript` as a multi-step conversational skill rather than building a desktop app
- **Why:** Target users are Claude Code PMs already in the terminal. Desktop app adds complexity without capability for this user. The friction is the import UX, not the view UX.
- **Rejected alternatives:** Electron/Tauri desktop app (overkill for current user; revisit if expanding to non-Claude-Code users)
- **Assumes:** The terminal multi-step flow is smooth enough to not feel like friction
- **Where it lives:** Discussed in conversation, not yet implemented

**Auto stakeholder memory, on-demand full synthesis**
- **Decided:** After each transcript, auto-run stakeholder memory update (cheap, targeted). Full project synthesis runs on demand, not automatically.
- **Why:** Full synthesis with Sonnet on a 6-stakeholder project costs ~$0.10–0.20. Running it after every transcript would compound cost quickly. Stakeholder memory update is scoped and cheap.
- **Rejected alternatives:** Auto-run everything (too expensive); run nothing automatically (loses the "magic" of the loop)
- **Assumes:** Users will remember to run synthesis periodically; a prompt at end of transcript add helps
- **Where it lives:** To be implemented in `/_add-transcript` skill

**Sonnet 4.6 for testing phase**
- **Decided:** Use `claude-sonnet-4-6` for all synthesis during testing
- **Why:** Key test metric is "did you learn something you didn't know?" — Sonnet quality matters more than cost savings during validation. Opus is overkill for structured extraction. Switch to Haiku-first only after quality is validated.
- **Rejected alternatives:** Haiku (cheaper but may not surface subtle insights); Opus (unnecessary for extraction tasks)
- **Assumes:** Testing budget can absorb ~$0.15–0.25 per full cycle

**Separate API key + stats.json for cost visibility**
- **Decided:** Dedicated Anthropic API key named "stakeholder-intelligence" + stats.json updated by skills after each run
- **Why:** Token costs will compound. Need visibility from day one. Cloudflare dashboard tracks per-key usage; stats.json gives in-context visibility via status bar.
- **Where it lives:** `context-library/memory/stats.json`, `si-status.sh`, `.claude/settings.json`
